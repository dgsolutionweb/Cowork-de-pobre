## Story Statement

Como usuário, quero buscar conteúdo nos arquivos do meu projeto usando pesquisa semântica com RAG, para encontrar informações relevantes sem precisar lembrar o nome exato do arquivo.

## Acceptance Criteria

1. **Busca por projeto ativo** - Sistema busca chunks indexados apenas do projeto ativo
2. **Exibição de resultados** - Resultados mostram arquivo, trecho relevante e score de relevância
3. **Highlight de matches** - Termos buscados são destacados nos trechos retornados
4. **Limite configurável** - Usuário pode ajustar quantidade de resultados (default: 10)
5. **Integração com assistente** - Resultados podem ser injetados no contexto do assistente
6. **Feedback visual** - Loading state durante busca, empty state quando sem resultados

## Scope

**IN:**
- Página de Pesquisa dedicada em `/research`
- Input de busca com debounce (300ms)
- Lista de resultados com arquivo + trecho + score
- Botão "Usar no contexto" para injetar no assistente
- Filtro por tipo de arquivo (.md, .txt, .pdf, .docx)

**OUT:**
- Pesquisa em múltiplos projetos simultaneamente
- Embeddings vetoriais (fase 2)
- Exportação de resultados

## Dependencies

- ✅ `project_chunks` table (FTS5)
- ✅ `ProjectIndexingService.search()`
- ✅ `projects:search-context` IPC handler
- ✅ RAG injection no `assistant:chat`
